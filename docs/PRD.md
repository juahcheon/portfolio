## 1. 한 줄 요약

**Windows 데스크톱을 메타포로 한 인터랙티브 포트폴리오.**  
정적 GitHub Pages(`juahcheon.github.io`) 경험을 Next.js 기반으로 재구성하고, 바탕화면·작업 표시줄·시작 메뉴·앱 창으로 이력·프로젝트를 탐색한다.

---

## 2. 배경과 동기

### 2.1 왜 Windows 데스크톱을 카피했는가

- **가장 친숙한 화면**을 웹으로 구현해 보고 싶었다. 대부분의 사용자가 하루 종일 보는 UI라, 익숙함 자체가 포트폴리오의 첫인상이 된다.
- **디테일을 맞추는 과정**에서 재미를 느꼈다. 아이콘 선택·정렬, 시작 메뉴 스크롤, 작업 표시줄 시계, 탐색기 헤더 등 작은 요소 하나하나가 “완성도”로 이어진다.
- **에셋 수집**은 쉽지 않았다. 레거시 `juahcheon.github.io`의 webp·PNG, 아이콘 세트, 작업 표시줄 이미지를 찾고 맞추는 과정이 있었고, 맞았을 때 만족감이 컸다.
- **프론트엔드 역량**을 한 화면에 모아 보여 줄 수 있다. 레이아웃, 상태(열린 창), SCSS 모듈, API 연동, 접근성(키보드)까지 면접에서 설명하기 좋은 단위다.

### 2.2 레거시와의 관계

| 구분 | `juahcheon.github.io` | `portfolio` (본 repo) |
|------|------------------------|------------------------|
| 형태 | 정적 페이지 + 레거시 데스크톱 스크립트 | Next.js 15 + API 분리 |
| 콘텐츠 | HTML/JS에 박혀 있음 | `api/data/portfolio.json` |
| 목표 | 온라인 이력서 | 동일 콘텐츠 + 구조 개선 + 유지보수 |

**성공 기준 (1차)**

- 배포 URL에서 데스크톱·작업 표시줄·시작 메뉴가 “Windows처럼” 느껴진다.
- 바탕화면 아이콘 더블클릭 → 해당 섹션 창이 **1클릭 이내**로 열린다.
- 면접에서 **3분 안에** “왜 Windows인지 / 어떤 창이 무엇인지” 설명할 수 있다.
- `npm run verify`로 web·api 빌드가 통과한다.

## 4. 핵심 사용자 시나리오

실제 Windows 화면과 **유사하게** 보이도록 노력했다. 다만 포트폴리오 목적상 **클릭 수는 최소화**한다 (아래 5절).

### 4.1 바탕화면

1. 사용자가 페이지에 진입하면 Windows 바탕화면과 유사한 레이아웃(벽지, 아이콘 열)을 본다.
2. 아이콘 **한 번 클릭** → 선택 하이라이트 (Windows와 동일).
3. 아이콘 **더블클릭** 또는 **Enter / Space** → 해당 앱 창 열림 또는 외부 링크(새 탭).
4. 빈 바탕화면 클릭 → 선택 해제.

### 4.2 작업 표시줄 (Lnb)

1. **Windows 버튼** → 시작 메뉴 토글.
2. 열린 창 목록 → 클릭 시 포커스.
3. 시계·트레이 아이콘(와이파이, 볼륨 등) → 시각적 장식(기능 없음, 분위기 유지).
4. 시작 메뉴에서 GitHub·타임라인·Cursor 등 바로 실행.

### 4.3 창

1. 창 제목 표시줄: 최소화·최대화·닫기.
2. 탐색기 스타일: **휴지통**, **내 PC** (폴더·드라이브 UI, 콘텐츠 탐색은 장식).
3. Office 메타포: **Word** → 소개(About), **제어판** → 스킬.
4. **Chrome** → iframe 모달, **GitHub** → Chrome 스타일 모달 + 프로필.

### 4.4 의도적으로 없는 시나리오

- 바탕화면 → 앱 → **그 앱 안에서 또 폴더를 눌러 들어가는** 다단계 탐색은 **없음**.
- “내 PC” 안의 폴더·드라이브는 **비주얼 재현**이며, 실제 파일 시스템과 연결되지 않음.
- 시작 메뉴의 일부 앱 행(Discord, Docker 등)은 **플레이스홀더**(클릭해도 동작 없음).

## 5. 제품 원칙

| 원칙 | 설명 |
|------|------|
| **친숙함 우선** | Windows 10/11 데스크톱·작업 표시줄·시작 메뉴 레이아웃을 따른다. |
| **클릭 최소화** | 이력서 정보는 **한 번의 더블클릭**으로 해당 창에 도달. 앱 내부 네비게이션은 두지 않는다. |
| **콘텐츠와 UI 분리** | 문구·경력·프로젝트는 API JSON; UI는 web에서 소비. |
| **점진적 디테일** | 1차는 “보이고 열리는 것”까지. 픽셀 퍼펙트·모든 앱 동작은 2차 이후. |

## 6. 기능 범위

### 6.1 바탕화면 아이콘 ↔ 콘텐츠 매핑 (1차)

| 바탕화면 라벨 | `windowId` / 동작 | 창 `kind` | 콘텐츠 |
|---------------|-------------------|-----------|--------|
| 휴지통 | `trash` | `recycle` | 휴지통 탐색기 UI |
| 내 PC | `hero` | `thisPc` | 탐색기 “내 PC” UI (장식) |
| 스킬 | `skills` | `skills` | 스킬 탐색기 목록 |
| 자기소개 | `about` | `about` | Word 스타일 소개 문서 |
| Cursor | `cursor` | `cursor` | 연락·GitHub 안내 + Cursor 제작 안내 |
| GitHub | `github` | `github` | GitHub 프로필(Chrome 모달) |
| 프로젝트 | `projects` | `projects` | **Chrome 창 UI** + `projects` JSON 본문(iframe 대신 임베드) |
| DS Helper | `external` | — | `https://dshelper.kr/` 새 탭 |

### 6.2 시작 메뉴·기타

- 시작 메뉴 **타임라인** → `timeline` 창 (학력·경력 연표).
- **Excel / PowerPoint 창은 사용하지 않음** (제거됨). 경력·프로젝트 상세는 JSON에 남을 수 있으나 전용 Office 창으로 노출하지 않음.

### 6.3 스코프 인 (1차)

- 데스크톱 + 작업 표시줄 + 시작 메뉴 + 다중 창(Zustand)
- API `GET /v1/portfolio`
- 레거시 에셋 동기화 (`npm run sync:legacy`)
- 키보드: 아이콘 Enter/Space, 시작 메뉴 Esc

### 6.4 스코프 아웃 (1차)

- 실제 파일 시스템·다운로드 폴더 연동
- 로그인·CMS 관리 화면
- 모바일 전용 UX (데스크톱 메타포 유지)
- 뚜레쥬르·탐앤탐스·KINNI 등 **제거된** 레거시 브랜드 아이콘
- **Excel / PowerPoint** Office 메타포 창
- **escapeFinal** 링크·아이콘

## 8. 면접관용 요약 (30초)

> “Windows 데스크톱을 그대로 따라 하기보다, **익숙한 UI로 포트폴리오를 탐색**하게 만든 프로젝트입니다. 바탕화면 아이콘은 Office·시스템 앱 이름으로 이력서 섹션에 매핑하고, **한 번 더블클릭**이면 내용 창이 열립니다. 콘텐츠는 API JSON으로 분리해 수정·배포를 나눴고, UI 디테일(시작 메뉴, 작업 표시줄)은 구현 과정에서 즐기면서 맞췄습니다.”
