Troubleshooting 1
발단

백엔드가 테스트/운영 서버로 이원화되어 있었는데, 처음엔 그냥 .env에서 URL을 직접 바꿔가며 쓰고 있었어요.

문제 사항

운영·Admin·테스트·로컬 네 환경을 오가면서 배포할 때마다 .env를 손으로 고치다 보니 실수가 잦았습니다.

해결

호스트명으로 현재 환경을 자동 판별하는 유틸 함수를 만들었습니다. 배포 도메인엔 운영 API를, 그 외엔 테스트 API를 자동으로 붙이고 소셜 로그인 리다이렉트 URI도 같은 방식으로 처리했습니다.

Troubleshooting 2
발단

쓰레기통 위치 지도에서 핀마다 실제 사진을 보여주는 기능을 붙였는데, 클릭할 때마다 이미지가 한 박자씩 느리게 떴습니다.

문제 사항

파악해보니 Next.js의 /_next/image 파이프라인을 타면서 외부 이미지 변환에 시간이 걸리고 있었습니다.

해결

해당 컴포넌트는 최적화 경로를 우회해 원본 URL을 바로 쓰도록 바꾸고, React Query 캐싱을 얹어 같은 데이터를 재요청할 땐 네트워크를 거치지 않도록 개선했습니다.

Troubleshooting 1
발단

Chrome, Explorer, Word 세 창 타입을 만들다 보니 각각 UI 구조가 제법 달랐고, 상태도 따로 관리하고 싶었습니다.

문제 사항

어느 창이 열려 있는지, 포커스가 어디 있는지 같은 UI 상태를 한 곳에서 다루다 보니 서버 데이터와 뒤섞이면서 관리가 점점 복잡해졌습니다.

해결

창 열림·포커스는 Zustand, 창 안의 프로젝트·스킬 데이터는 React Query로 역할을 나눴습니다. 성질이 다른 상태를 분리하고 나니 각자의 책임이 훨씬 명확해졌습니다.
